## Files Detail

### Configuration & Documentation

- **CLAUDE.md** - PM agent's complete startup guide covering system overview, agent roles, behavior rules (don't ask terminal questions, propose plans, update Trello), token cost management strategy (use cheapest model for task), task routing table, Trello integration details, Telegram communication protocol, and error recovery procedures

- **README.md** - User-facing documentation with quick start guide, prerequisites, setup instructions, system architecture diagram showing Founder → Telegram Bridge → Navi → Agents workflow, Telegram commands, Trello board structure, and repo directory layout

- **VISION.md** - Strategic roadmap describing current Phase 0 architecture (Claude Max on local machine), planned Phase 1 (AWS cloud deployment with FastAPI, PostgreSQL, Weaviate), and Phase 2 (SaaS platform), including cost models and limitations

- **NAVAIA-BUILD-INSTRUCTIONS.md** - Detailed implementation guide for development, deployment, configuration, and troubleshooting of the workforce system

- **.env** - Secrets file (not committed) with API credentials for Telegram bot, Trello, GitHub, AWS, and other services needed for integrations

- **.env.example** - Public template showing the structure and names of required environment variables without revealing actual secrets

- **.gitignore** - Exclusion rules preventing secrets (.env), cache (__pycache__, .pyc), dependencies (node_modules), logs, and ephemeral files (workspace/tasks/inbox, workspace/comms) from being committed while preserving .gitkeep files

## Directories Detail

### agents/
Contains CLAUDE.md identity files for each of the 4 team agents:
- **pm/** - Navi (PM/Team Lead) instructions
- **creative/** - Muse (Creative & Marketing) instructions
- **technical/** - Arch (Technical/Engineer) instructions
- **admin/** - Sage (Admin & Finance) instructions

Each agent reads their corresponding CLAUDE.md file on startup to understand their role, skills, and responsibilities.

### knowledge/
Company knowledge base organized by function:
- **INDEX.md** - Auto-generated catalog of all knowledge files (regenerated by catalog.py)
- **company/** - Pitch deck, brand guide, team bios, mission statement
- **sales/** - Proposals, pricing sheets, case studies, customer testimonials
- **products/** - Product documentation, feature specs, user guides
- **finance/** - Budget forecasts, invoices, financial statements, burn rate analysis
- **legal/** - Contracts, NDAs, terms of service, compliance docs
- **marketing/** - Campaign plans, content calendar, social media assets
- **technical/** - Architecture docs, API specifications, infrastructure details
- **hr/** - Job descriptions, policies, employee handbooks
- **templates/** - Reusable document templates for contracts, emails, proposals

### workspace/
Live working directory for active agent work:
- **tasks/inbox/** - New tasks from Founder (via Telegram bridge)
- **tasks/active/** - Tasks currently being worked on by agents
- **tasks/done/** - Completed tasks moved here after approval
- **tasks/rejected/** - Tasks rejected by Founder and awaiting rework
- **outputs/creative/** - Content outputs from Muse (emails, social posts, campaigns)
- **outputs/technical/** - Code, deployment configs, and architecture docs from Arch
- **outputs/admin/** - Documents, proposals, financial analysis from Sage
- **comms/to-founder/** - Messages queued for delivery to Founder via Telegram
- **comms/from-founder/** - Replies and approvals received from Founder
- **comms/inter-agent/** - File handoffs between team members
- **bot.log** - Telegram bridge activity log
- **crew.log** - Agent team execution log

### scripts/
Automation and lifecycle management:
- **setup.sh** - First-time environment setup (creates directories, installs dependencies, configures .env)
- **start.sh** - Launches a single agent session with task pickup and execution
- **loop.sh** - 24/7 operation loop that restarts the crew whenever work arrives (prevents token waste by exiting when idle)

### tools/
Integration and utility tools:
- **telegram_bridge.py** - Python bridge that monitors Telegram bot inbox, forwards new tasks as files to workspace/tasks/inbox/, and watches workspace/comms/to-founder/ to send messages back to Founder
- **trello_api.sh** - Bash helper functions for Trello REST API operations (create cards, move between lists, update status, add labels)
- **catalog.py** - Python script that auto-scans knowledge/ directory and regenerates knowledge/INDEX.md with file descriptions and locations
- **requirements.txt** - Python package dependencies (python-telegram-bot, requests, etc.)

## Key Files for Development

| Task | Key Files |
|------|-----------|
| Add a new knowledge file | Add to `knowledge/{category}/`, then run `python3 tools/catalog.py` |
| Modify agent behavior | Edit `agents/{role}/CLAUDE.md` |
| Add a new integration | Update `tools/`, add credentials to `.env`, reference in agent CLAUDE.md |
| Debug agent execution | Check `workspace/crew.log` and `workspace/bot.log` |
| Add startup steps | Edit `scripts/setup.sh` or `scripts/start.sh` |
| Control Trello from scripts | Source `tools/trello_api.sh` and call functions |
| Send messages to Founder | Write markdown file to `workspace/comms/to-founder/` |
